# Ansible Integration Guide

## Overview

Diese Docker-Lösung ist speziell für die Integration mit Ansible-Rollen konzipiert. Das Ziel: **Ansible orchestriert, Docker führt aus**.

## Ansible Task Structure

### 1. Deployment Task

```yaml
---
- name: Deploy Solr Docker Solution
  block:
    - name: Create deployment directory
      file:
        path: "/opt/solr-{{ customer_name }}"
        state: directory
        owner: root
        group: root
        mode: '0755'

    - name: Copy Docker Compose files
      copy:
        src: "{{ role_path }}/solr-moodle-docker/"
        dest: "/opt/solr-{{ customer_name }}/"
        owner: root
        group: root
        mode: preserve

    - name: Generate .env file
      template:
        src: env.j2
        dest: "/opt/solr-{{ customer_name }}/.env"
        owner: root
        group: root
        mode: '0600'

    - name: Generate configuration
      command: ./scripts/generate-config.sh
      args:
        chdir: "/opt/solr-{{ customer_name }}"
      register: config_gen
      changed_when: true

    - name: Start Solr (monitoring mode: {{ monitoring_mode }})
      command: >
        docker compose
        {% if monitoring_mode == 'exporter-only' %}--profile exporter-only{% endif %}
        {% if monitoring_mode == 'full' %}--profile monitoring{% endif %}
        up -d
      args:
        chdir: "/opt/solr-{{ customer_name }}"
      register: docker_up

    - name: Wait for Solr to be healthy
      uri:
        url: "http://localhost:{{ health_api_port }}/health"
        method: GET
        return_content: yes
        status_code: 200
      register: health_check
      until: health_check.json.status == "healthy"
      retries: 30
      delay: 2

    - name: Get deployment status
      uri:
        url: "http://localhost:{{ health_api_port }}/health"
        method: GET
        return_content: yes
      register: final_status

    - name: Display deployment summary
      debug:
        msg:
          - "Deployment successful!"
          - "Customer: {{ final_status.json.customer }}"
          - "Version: {{ final_status.json.version }}"
          - "Solr Status: {{ final_status.json.solr.status }}"
          - "Cores: {{ final_status.json.cores | length }}"
          - "Memory Usage: {{ final_status.json.system.memory.usage_percent }}%"

  rescue:
    - name: Get error details
      command: docker compose logs
      args:
        chdir: "/opt/solr-{{ customer_name }}"
      register: docker_logs
      ignore_errors: yes

    - name: Display error logs
      debug:
        msg: "{{ docker_logs.stdout_lines }}"

    - name: Fail with detailed error
      fail:
        msg: "Solr deployment failed. Check logs above."
```

### 4. Health Check Task

```yaml
---
- name: Check Solr Health
  uri:
    url: "http://localhost:{{ health_api_port }}/health"
    method: GET
    return_content: yes
    status_code: 200
  register: health_response
  failed_when: health_response.json.status != "healthy"

- name: Assert Solr is healthy
  assert:
    that:
      - health_response.json.status == "healthy"
      - health_response.json.solr.available == true
      - health_response.json.cores | length > 0
    fail_msg: "Solr is not healthy: {{ health_response.json }}"
    success_msg: "Solr is healthy and running"

- name: Store deployment facts
  set_fact:
    solr_deployment:
      customer: "{{ health_response.json.customer }}"
      version: "{{ health_response.json.version }}"
      status: "{{ health_response.json.status }}"
      cores: "{{ health_response.json.cores }}"
      memory_usage: "{{ health_response.json.system.memory.usage_percent }}"
      uptime: "{{ health_response.json.system.uptime_seconds }}"
```

## Example Playbook

```yaml
---
- name: Deploy Solr with Docker
  hosts: solr_servers
  become: yes

  vars:
    customer_name: "acme-corp"
    moodle_app_domain: "moodle.acme.com"
    monitoring_mode: "exporter-only"

    # Secure passwords from Ansible Vault
    solr_admin_password: "{{ vault_solr_admin_password }}"
    solr_support_password: "{{ vault_solr_support_password }}"
    solr_customer_password: "{{ vault_solr_customer_password }}"

    # Remote monitoring
    remote_prometheus_url: "https://prometheus.company.com/api/v1/write"
    exporter_bind_ip: "0.0.0.0"

  tasks:
    - name: Include Solr role
      include_role:
        name: solr
        tasks_from: docker-deploy

    - name: Verify deployment
      include_role:
        name: solr
        tasks_from: health-check

    - name: Report to monitoring
      uri:
        url: "{{ monitoring_webhook }}"
        method: POST
        body_format: json
        body:
          event: "solr_deployed"
          customer: "{{ customer_name }}"
          status: "{{ solr_deployment.status }}"
          version: "{{ solr_deployment.version }}"
      when: monitoring_webhook is defined
```

## Error Handling

### Ansible Error Scenarios

```yaml
- name: Deploy with error handling
  block:
    - name: Deploy Solr
      include_tasks: deploy.yml

  rescue:
    - name: Check what failed
      uri:
        url: "http://localhost:{{ health_api_port }}/health"
        method: GET
        return_content: yes
        status_code: [200, 503]
      register: error_health
      ignore_errors: yes

    - name: Get Docker logs
      command: docker compose logs --tail=100
      args:
        chdir: "{{ solr_docker_dir }}"
      register: docker_logs

    - name: Save error state
      copy:
        content: |
          Deployment failed at {{ ansible_date_time.iso8601 }}

          Health API Response:
          {{ error_health.json | to_nice_json }}

          Docker Logs:
          {{ docker_logs.stdout }}
        dest: "{{ solr_docker_dir }}/error-{{ ansible_date_time.epoch }}.log"

    - name: Attempt rollback
      command: docker compose down
      args:
        chdir: "{{ solr_docker_dir }}"
      ignore_errors: yes

    - name: Fail with details
      fail:
        msg: "Deployment failed. Error log: {{ solr_docker_dir }}/error-{{ ansible_date_time.epoch }}.log"

  always:
    - name: Send notification
      # Your notification logic here
      debug:
        msg: "Deployment {{ 'succeeded' if ansible_failed_result is not defined else 'failed' }}"
```

```bash

# Content:

vault_solr_customer_password: "customer_password_789"

```

### 2. Idempotent Deployments

```yaml
- name: Check if already deployed
  stat:
    path: "{{ solr_docker_dir }}/.deployed"
  register: deployment_marker

- name: Deploy only if needed
  include_tasks: deploy.yml
  when: not deployment_marker.stat.exists or force_redeploy | default(false)

- name: Create deployment marker
  file:
    path: "{{ solr_docker_dir }}/.deployed"
    state: touch
  when: solr_deployment.status == "healthy"
```

## Troubleshooting

### Health API not responding

```yaml
- name: Debug health API
  command: docker logs {{ customer_name }}_health_api
  register: health_api_logs

- name: Check if port is listening
  wait_for:
    port: "{{ health_api_port }}"
    timeout: 5
  register: port_check
  ignore_errors: yes

- name: Display troubleshooting info
  debug:
    msg:
      - "Health API logs: {{ health_api_logs.stdout_lines }}"
      - "Port {{ health_api_port }} listening: {{ port_check.failed | ternary('No', 'Yes') }}"
```

### Solr not starting

```yaml
- name: Check Solr container
  command: docker inspect {{ customer_name }}_solr
  register: solr_inspect

- name: Check init container logs
  command: docker logs {{ customer_name }}_solr_init
  register: init_logs

- name: Display diagnostics
  debug:
    msg:
      - "Container state: {{ (solr_inspect.stdout | from_json)[0].State.Status }}"
      - "Init logs: {{ init_logs.stdout_lines }}"
```

## Summary

**Ansible-Rolle:**
- Deployed Docker-Lösung
- Generiert `.env` aus Templates
- Startet Docker Compose mit richtigen Profiles
- Wartet auf Health API
- Verifiziert Status
- Meldet Erfolg/Fehler zurück

**Docker-Lösung:**
- Führt Solr aus
- Stellt Health API bereit
- Exponiert Metriken (optional)
- Managed eigene Lifecycle

**Ergebnis:**
- Klare Trennung der Verantwortlichkeiten
- Einfache Integration
- Robuste Fehlerbehandlung
- Skalierbar und wartbar
